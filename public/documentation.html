Documentation

Get Endpoints

Endpoint Method URL Response Parameters
/ GET localhost:8080 Static HTML N/A
/documentation GET http://localhost:8080/documentation Static HTML N/A
/movies GET http://localhost:8080/movies JSON /:title
/featured GET http://localhost:8080/featured JSON N/A
/directors GET http://localhost:8080/directors JSON /:name
/users GET http://localhost:8080/users JSON N/A
/favorites GET http://localhost:8080/favorites JSON /:id

Put Endpoints

Endpoint Method URL Response Parameters
/favorites PUT http://localhost:8080/favorites String /:id/:title
/users PUT http://localhost:8080/users String /:name/:newName

Post and Delete Endpoints

Endpoint Method URL Response Parameters
/users POST http://localhost:8080/users String { "name", "email" }
/users DELETE http://localhost:8080/users String /:email

String Responses

String responses will announce either the current issue or the success of your request. All requests are accompanied by a corresponding HTML status code.

Example 1: Odd Thomas added to your favorites.
Example 2: The username "watcher" is already taken.

PUT to favorites

If you PUT a movie title to your favorites list that does not already exist in the list, it will be saved to favorites.
PUT that movie to your favorites again to remove it from the list. This functions like a toggle.
